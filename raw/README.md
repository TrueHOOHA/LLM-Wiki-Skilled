# 原始来源策略

## 存放内容

`raw/` 目录包含您不可变的源文档。这些是真实来源。大语言模型从中读取但绝不会修改它们。

* `raw/sources/` — 基于文本的来源：文章、论文、转录稿、会议记录、书籍章节等。

* `raw/assets/` — 二进制或媒体文件：图像、图表、数据文件、PDF 等。

## 不可变规则

**一旦文件放入** **`raw/`，就不得对其进行编辑。** 如果您有修订版的来源，请将其作为新文件添加（例如 `article-v2.md`），然后重新摄入。这样可以保留知识的溯源链。

## 命名规范

* 使用描述性的 kebab-case 文件名。

* 对于时间相关的来源，添加日期前缀：`YYYY-MM-DD--article-title.md`。

* 示例：

  * `2026-05-15--attention-is-all-you-need.md`

  * `2026-05-15--meeting-transcript-project-kickoff.md`

  * `report-on-market-trends-2026.md`

## 添加来源

1. 将来源文件保存或剪贴到 `raw/sources/`。
2. （可选）将引用的图片下载到 `raw/assets/`。
3. 告知您的大语言模型代理来摄入它。
4. 代理将读取、总结并将其整合到知识库中。

## 资源处理

* 来源引用的图片应以描述性名称下载到 `raw/assets/`。

* 大语言模型可以单独查看图片（内联图片读取并不总是可靠）。

* 使用相对路径从来源页面链接到资源：`../assets/image-name.png`。
