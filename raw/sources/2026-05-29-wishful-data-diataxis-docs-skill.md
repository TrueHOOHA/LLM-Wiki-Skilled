Captured page summary/text:

Diátaxis is described as "A systematic approach to technical documentation authoring." It is presented as a way of thinking about and doing documentation, prescribing approaches to content, architecture, and form that emerge from understanding documentation users' needs.

The framework identifies four distinct needs and four corresponding forms of documentation:

- tutorials
- how-to guides
- technical reference
- explanation

The site says documentation should be organized around those needs. It claims Diátaxis solves problems related to documentation content, style, and architecture; is lightweight and implementation-agnostic; and helps maintainers think effectively about documentation quality.
