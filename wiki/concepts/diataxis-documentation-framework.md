# Diátaxis Documentation Framework

## Definition
Diátaxis Documentation Framework is a technical-documentation architecture that separates documentation by the user's need: tutorials support guided skill acquisition, how-to guides support goal-oriented work, reference supports accurate technical lookup, and explanation supports understanding-oriented reflection.

## Scope
In Tolaria, this concept applies to documentation structure, Hermes skill authoring, and knowledge-note organization. It is not a claim that every skill must have four separate files or headings. The practical lens is diagnostic: identify whether a passage is trying to teach, guide work, describe machinery, or explain context, then keep the form aligned with that need. [[DataThoughts tweet on Diátaxis docs skill]] supplies the weak social discovery route; the Diátaxis site supplies the stronger primary framework evidence.

## Contrasts
- Versus a generic docs checklist: Diátaxis classifies documentation by user situation and purpose, not by surface format.
- Versus top-down reorganization: Diátaxis recommends small, responsive improvements that let structure emerge from better-serving user needs.
- Versus [[Skillify]]: Skillify turns recurring work into durable agent behavior, while Diátaxis helps decide whether the resulting skill content is tutorial-like, how-to-like, reference-like, or explanation-like.
- Versus [[LLM Wiki Pattern]]: LLM Wiki governs source-backed knowledge layers and provenance, while Diátaxis helps shape the user-facing form of those notes.

## Evidence
- [[DataThoughts tweet on Diátaxis docs skill]] links the weak social claim to the Diátaxis primary documentation site and records the four modes plus the lack of evidence for the unspecified docs skill.
- The Diátaxis homepage describes the framework as a systematic approach to documentation content, style, and architecture around user needs.
- The Diátaxis primer defines tutorials as lessons, how-to guides as directions for competent users at work, reference as neutral technical description, and explanation as context/background for understanding.
- The Diátaxis workflow page recommends using the framework as a guide rather than a plan: choose a small piece of documentation, assess the user need it serves, make one improvement, and repeat.
