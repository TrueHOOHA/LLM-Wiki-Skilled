# DataThoughts tweet on Diátaxis docs skill

## Summary
DataThoughts claims in an X reply that they made a documentation skill following the Diátaxis framework and that a user can ask an agent to make a new skill. The tweet is weak evidence for the existence or quality of that unspecified skill because the accessible capture exposes no skill file, repository, prompt, transcript, or before/after documentation artifact. The durable value is the linked Diátaxis documentation framework itself: a primary documentation site that distinguishes tutorials, how-to guides, reference, and explanation by user need. For Tolaria and [[Hermes Agent]], this source is useful as a documentation-quality lens for skills and knowledge notes, not as permission to patch skills or restructure docs.

## Key Claims
1. The social post claims a practitioner made a docs skill that follows Diátaxis, but provides no inspectable implementation evidence.
2. The linked Diátaxis site defines four documentation modes: tutorials for guided learning, how-to guides for goal-oriented work, reference for accurate technical description, and explanation for understanding-oriented context.
3. Diátaxis frames documentation quality around user need: action vs cognition, and acquisition of skill vs application of skill.
4. Diátaxis recommends pragmatic, small-step improvement rather than top-down restructuring: inspect one page or paragraph, identify its user need, make one improvement, then repeat.
5. Applied to Hermes skill authoring, the useful implication is separation of modes inside a skill or documentation set: a skill may need concise how-to procedure, reference-like command contracts, and explanation of tradeoffs, but should not blur those modes without intent.
