# Nick Radford tweet on ffmgen AI FFmpeg command generator

## Summary
Nick Radford's tweet says, "Or use https://ffmgen.nickradford.dev to generate the ffmpeg command," pointing to [[ffmgen]], a web utility that asks users to describe video/audio work and returns an FFmpeg command. The source is weak/social evidence: Alpha observed that the page and `/api/generate` endpoint exist, but did not establish command accuracy, command safety, public source-code review, or robustness of the prompt-injection filtering. The durable Tolaria value is not product adoption; it is a concrete example for [[AI Command-generation Safety Pattern]], especially review-before-run warnings, quoting/escaping, dry-run or local verification, destructive-operation labeling, and skepticism toward client-side-only injection filters.

## Key Claims
1. The tweet claims [[ffmgen]] can generate FFmpeg commands from natural-language video/audio tasks.
2. The linked page claims users can "Describe what you want to do with your video or audio" and "Get the exact ffmpeg command instantly."
3. The page visibly warns: "Commands are generated by AI. Always review before running."
4. Alpha's source check observed a Next.js/Vercel app, `/api/generate` returning event-stream output, and model metadata for `openai/gpt-oss-20b`.
5. Alpha observed client-side validation for empty prompts, prompts over 500 characters, and several prompt-injection phrases, but this is only an affordance, not proof of safety.
6. Accuracy, safety, robustness of filters, shell quoting quality, destructive-operation handling, and source-code review were not established.
